# Vuex

核心概念：

  • State
    • 容器的数据，类似于组件中的 data
  • Getter
    • 容器的计算属性，类似于组件的 computed
  • Mutation
    • 一个函数，用来修改 state 数据
    • 注意1：请求务必使用 mutation 来修改数据，这样做的目的主要有两个：
      • 第一主要是容器的状态在外部被修改会导致数据管理的混乱
      • 第二是只有 mutation 修改数据才会在 DevTools 中产生数据改变的历史记录，可以拥有 Time Travel 的调试体验
    • 注意2：不要在 mutation 中执行异步操作修改 state 数据
      • 主要原因是异步操作修改 state 不会在调试工具中产生数据改变的历史记录
  • Action
  • Module

没有异步操作的数据修改流程：

  • 在组件中 commit 提交 mutation
  • 在 mutation 中修改数据状态
  • 装填改变，重新渲染视图

有异步操作的数据修改流程：

  • 在组件中 dispatch 发起 action
  • 在 action 中执行异步操作
  • 在 action 中异步操作执行结束，提交 mutation 修改 state
  • state 发生改变，重新渲染视图更新
