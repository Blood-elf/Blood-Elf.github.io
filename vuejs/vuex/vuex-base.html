# Vuex 教程

# 介绍

视频：Vuex-介绍

目标：

  • 了解什么是 Vuex
  • 了解 Vuex 的使用场景
  • 回顾组件通信
    • 父传子：Props
    • 子传父：Events
    • 非父子：Event Bus
    • 。。。
  • Vuex 是什么？
    • Vuex 是 Vue 官方开发的一个 Vue 核心插件
    • 它的作用是进行状态管理，解决复杂组件通信
  • 什么情况下使用 Vuex？
    • 当我们的应用遇到 多个组件共享状态 时
      • 多个视图依赖于同一状态
      • 来自不同视图的行为需要变更同一状态
    • 最典型的就是类似于购物车这样的复杂交互场景
    • 注意：没有以上需求的场景不要使用 Vuex 来管理数据，反而让你的应用变得繁琐冗余。。。
    • 提示：一般都会在项目中使用 Vuex，但不是全部由 Vuex 来管理组件状态
      • 我们会把满足以上条件的场景来使用 Vuex 来管理
      • 不满足或者说不需要还是在组件内部进行管理
    • 当然，如果项目中一个类似的场景都没有，可以完全不用。

# 安装和配置

视频：Vuex-快速入门-安装和配置

目标：

  • 怎么下载 Vuex？
  • 怎么配置 Vuex？
  • 安装 Vuex 到项目中：npm i vuex
  • 在项目中新建 store/index.js 并写入以下内容
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

/**
 * 创建一个 Vuex 容器实例，用来在组件的外部管理共享的数据状态
 */
const store = new Vuex.Store({
  /**
   * 类似于组件中的 data
   */
  state: {
  },

  /**
   * 类似于组件中的 methods
   * 作用：修改 state
   * mutation 函数默认接收一个参数：state 数据对象
   * 1. 定义 mutation 函数
   * 2. 在组件中调用 mutation
   */
  mutations: {
  },

  /**
   * 类似于 mutation
   * 作用：执行异步操作，提交 mutation 修改 state
   */
  actions: {}
})

export default store

# Mutation

视频：Vuex-快速入门-Mutation

目标：

  • 了解 Mutation 的作用是什么
    • 修改 state 数据的
  • 掌握如何定义 Mutation
    • 一个函数
    • 函数接收一个默认参数：state（就是容器中的 state）
  • 掌握如何调用 Mutation
    • store.commit('mutation名称')

总结：

  • 修改数据使用 mutation
  • mutation 的第1个参数是 state 对象
  • 注意：只有 mutation 才会被调试工具观测记录到，任何其他非 mutation 方式修改数据都会反应到调试工具中，所以务必永远通过 mutation 修改 state 数据状态

# Action

视频：Vuex-快速入门-Action

目标：

  • 了解 Action 的作用是什么
    • 类似于 mutation，执行异步操作
  • 掌握如何定义 Action
  • 掌握如何调用 Action

注意：不要在 mutation 中执行异步操作修改 state，或者说 mutation 中必须是同步代码

总结：

  • 不要在 mutation 中执行异步操作修改 state
  • 正确的做法是将异步操作写到 action 中
  • action 中异步操作结束以后，提交 mutation 来修改 state
  • 注意：也不要在 action 中直接修改 state，永远通过 mutation 来修改 state

# 管理流程

视频：Vuex-快速入门-管理流程

目标：

  • 了解 Vuex 的状态管理流程
    • 同步操作
    • 异步操作

同步操作：

  • commit
  • mutation
    • 注意：不要在 mutation 中执行异步操作修改 state，使用 action
  • state

异步操作：

  • dispatch
  • action
  • mutation
  • state

注意：

  • 只有在 mutation 中修改 state 才会被调试工具观测记录到
  • 任何 mutation 之外的修改 state 都不会被记录观测

# Mutation

视频：Vuex-核心API-Mutation

目标：

  • 了解 mutation 传参
  • 了解 Mutation 的提交方式

如何传参：

你可以向 store.commit 传入额外的参数，即 mutation 的 载荷（payload）：

// ...
mutations: {
  increment (state, n) {
    state.count += n
  }
}
store.commit('increment', 10)

在大多数情况下，载荷应该是一个对象，这样可以包含多个字段并且记录的 mutation 会更易读：

// ...
mutations: {
  increment (state, payload) {
    state.count += payload.amount
  }
}
store.commit('increment', {
  amount: 10
})

  increment (state, payload) {
    state.count += payload.amount

如何更方便的调用 action：

方式一：this.$store.commit('mutation函数名称', 可选的传递参数)

方式二：使用 mapMutations 辅助函数

import { mapMutations } from 'vuex'

export default {
  // ...
  methods: {
    ...mapMutations([
      'increment', // 将 `this.increment()` 映射为 `this.$store.commit('increment')`

      // `mapMutations` 也支持载荷：
      'incrementBy' // 将 `this.incrementBy(amount)` 映射为 `this.$store.commit('incrementBy', amount)`
    ]),
    ...mapMutations({
      add: 'increment' // 将 `this.add()` 映射为 `this.$store.commit('increment')`
    })
  }
}

# Getter

视频：Vuex-核心API-Getter

目标：

  • 了解什么是 Getter 及其作用
    • Getter 就是容器的计算属性
  • 掌握如何定义 Getter
  • 掌握如何访问 Getter
    • this.$store.geeters.xxx
    • mapGetters 辅助函数

定义 Getter：

const store = new Vuex.Store({
  state: {
    todos: [
      { id: 1, text: '...', done: true },
      { id: 2, text: '...', done: false }
    ]
  },
  getters: {
    doneTodos: state => {
      return state.todos.filter(todo => todo.done)
    }
  }
})
